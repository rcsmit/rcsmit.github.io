Tenant Config Manual

How to create a config file for a new event listing site on this platform.

Overview

Every site (tenant) is defined by a single JSON file: [tenantid].config.json.
This file controls everything — branding, data source, categories, labels, sharing, and more.
No code changes are needed to launch a new tenant; only config.js needs one line updated to point to the new file.

Step 1 — Name your tenant

tenant.name — CamelCase two-word compound, required

The tenant name must be a CamelCase compound of exactly two words joined together, with no space. Examples:

Good ✅ Bad ❌
AcroInsiders Acro Insiders (space)
BailaConmigo bailaconmigo (no capitals)
KaraokeEvents Karaoke_Events (underscore)
SalsaNights SalsaNightsWorld (three words)

Why: The loading overlay splits the name on the second capital letter to render it in two colours — e.g. Acro in white and Insiders in a dimmed tint. A name with no second capital, or with more than two parts, will display incorrectly.

tenant.id — lowercase, no spaces

Used internally in console logs and ICS file UIDs. Typically the name lowercased: acroinsiders, bailaconmigo.

Step 2 — Fill in the full tenant block

{
  "tenant": {
    "id": "acroinsiders",
    "name": "AcroInsiders",
    "tagline": "Find acroyoga events worldwide",
    "status": "active",
    "plan": "pro",
    "created_at": "2026-05-01",
    "custom_domain": "acroinsiders.com",
    "subdomain": "acroinsiders"
  }
}
Field Purpose
id Lowercase identifier, used in logs and ICS files
name Display name — must be CamelCase two-word compound (see above)
tagline Short description shown in the browser tab title
status active or inactive
plan For your own records; not used by the platform
created_at ISO date, for your own records
custom_domain Used to build deep links and the back-to-events URL
subdomain Fallback if custom_domain is not set

Step 3 — Branding

{
  "branding": {
    "colors": {
      "primary":         "#f19072",
      "secondary":       "#12807a",
      "background":      "#fef6f4",
      "text":            "#2a2a2a",
      "muted":           "#7a7a7a",
      "border":          "#e0d4cf",
      "primary_light":   "#fcdacf",
      "secondary_dark":  "#0d5f5a",
      "card_background": "#ffffff"
    },
    "fonts": {
      "heading": "Corben",
      "body":    "Josefin Sans"
    },
    "logo_url":    "https://yourdomain.com/logo.png",
    "favicon_url": "https://yourdomain.com/favicon.ico",
    "dark_mode":   false
  }
}

All color values are hex. They map to CSS variables used throughout the site:

Field CSS var Used for
primary --p1 Header background, accents, CTA hover
secondary --p2 Dates, links, secondary buttons
background --p8 Page background
card_background --p9 Event card and modal background
primary_light --p7 Hover tints, filter backgrounds
secondary_dark --p2-dark Darker secondary accents
muted --p5 Subdued text (locations, counts)
border --p6 Card borders, dividers

Fonts must be valid Google Fonts names. They are loaded automatically — no <link> tag needed. Use the exact name as it appears on fonts.google.com, e.g. "Josefin Sans", "Corben", "Righteous", "Nunito".

Step 5 — Categories

Each category gets an id, a display label, an emoji icon, and a colour used for badges.

{
  "categories": [
    { "id": "festival",   "label": "Festival",   "icon": "🎪", "color": "#f19072" },
    { "id": "retreat",    "label": "Retreat",    "icon": "🏕️", "color": "#12807a" },
    { "id": "intensive",  "label": "Intensive",  "icon": "🔥", "color": "#f59e0b" },
    { "id": "event",      "label": "Event",      "icon": "📍", "color": "#ef4444" },
    { "id": "training",   "label": "Training",   "icon": "🎓", "color": "#8b5cf6" }
  ]
}
  • The order here determines the order of filter pills in the UI.
  • label must exactly match what appears in the data spreadsheet's category column.
  • id is lowercase, used internally only.
  • Add or remove categories freely — the UI builds from this list.

Step 6 — Locations

{
  "locations": {
    "mode": "continents",
    "regions": [
      "Africa", "Asia", "Europe",
      "North America", "Oceania", "South America"
    ]
  }
}

regions defines which continent filter options appear. Values must match what is used in the spreadsheet's continent column.

Step 7 — Views

{
  "views": {
    "default":   "list",
    "available": ["list", "calendar", "map"],
    "list": {
      "page_size":                  9,
      "min_page_size":              9,
      "card_min_width_px":          320,
      "min_rows":                   3,
      "show_cat_picks":             true,
      "cat_picks_label":            "Cat's Picks",
      "cat_picks_field":            "sticky",
      "cat_picks_icon":             "⭐",
      "show_featured_image":        true,
      "show_description_truncated": true,
      "show_tags":                  false,
      "show_venue":                 true,
      "show_visible_count":         true
    },
    "calendar": {
      "start_day":               "monday",
      "show_empty_month_message": true
    },
    "map": {
      "default_lat":  20.0,
      "default_lon":  10.0,
      "default_zoom": 2
    }
  }
}

List view notes:

  • page_size — number of cards per page on desktop
  • min_page_size — minimum cards shown (used by the preview widget)
  • cat_picks_label — the heading shown above highlighted events (e.g. "Cat's Picks", "Staff Picks", "Miguel's Picks")
  • cat_picks_field — the spreadsheet column that marks an event as picked (truthy value = shown in picks strip)
  • show_featured_image — set false if your data has no images

Map view notes:

  • default_lat / default_lon — where the map centres on first load
  • default_zoom — 2 shows the whole world; increase to zoom in on a region

Step 8 — Submission

{
  "submission": {
    "enabled":                true,
    "requires_approval":      true,
    "duplicate_check":        true,
    "duplicate_check_fields": ["name", "start_date"],
    "notification_email":     "hello@yourdomain.com",
    "confirmation_message":   "Thanks! Your event will appear after review.",
    "allowed_submitters":     "anyone",
    "google_sheet_id":        "YOUR_SHEET_ID_HERE",
    "google_sheet_tab":       "data"
  }
}

Set enabled: false to hide the submit form entirely.

Step 9 — Sharing

{
  "sharing": {
    "whatsapp":           true,
    "whatsapp_template":  "Check out this event!\n{name}\n{start_date} – {end_date}\n📍 {city}, {country}\n{url}",
    "gcal":               true,
    "copy_link":          true,
    "copy_link_template": "https://yourdomain.com/show-event/?id={slug}"
  }
}

Available placeholders in whatsapp_template: {name}, {start_date}, {end_date}, {city}, {country}, {url}.

copy_link_template uses {slug} (auto-generated from event name + date). Set whatsapp, gcal, or copy_link to false to hide those buttons in the event modal.

Step 10 — Data source

{
  "data_source": {
    "type":               "sheets",
    "worker_url":         "https://your-worker.workers.dev/events",
    "fallback_type":      "csv",
    "fallback_sheet_id":  "YOUR_GOOGLE_SHEET_ID",
    "fallback_sheet_tab": "data",
    "columns": {
      "url":            "URL",
      "name":           "EVENT NAME",
      "venue":          "EVENT VENUE NAME",
      "start_date":     "EVENT START DATE",
      "end_date":       "EVENT END DATE",
      "all_day":        "ALL DAY EVENT",
      "timezone":       "TIMEZONE",
      "category":       "EVENT CATEGORY",
      "continent":      "CONTINENT",
      "tags":           "EVENT TAGS",
      "featured_image": "EVENT FEATURED IMAGE",
      "website":        "EVENT WEBSITE",
      "description":    "EVENT DESCRIPTION",
      "city":           "CITY",
      "state":          "STATE",
      "country":        "COUNTRY",
      "sticky":         "STICKY IN MONTH VIEW",
      "lat":            "LAT",
      "lon":            "LON",
      "insider_deals":  "INSIDER DEAL"
    }
  }
}
  • worker_url — the Cloudflare Worker endpoint that returns { rows: [...] }. Leave as "" if not using a worker; the platform will go straight to the CSV fallback.
  • fallback_sheet_id — the Google Sheets ID from the URL: docs.google.com/spreadsheets/d/THIS_PART/edit
  • fallback_sheet_tab — the tab name (default: data)
  • columns — maps internal field names to your actual spreadsheet column headers. The right-hand values must exactly match your sheet's header row, including capitalisation and spaces.

The sheet must be shared as "Anyone with the link can view".

Step 11 — Activate the tenant

In config.js, change the one line at the top:

const TENANT_CONFIG_FILE = 'yourtenant.config.json';

That's it. All scripts read TENANT_CONFIG_FILE on load — no other file needs touching.

Quick checklist

  • tenant.name is a CamelCase two-word compound (e.g. SalsaNights)
  • tenant.custom_domain is set (used for deep links and back-to-events URL)
  • All categories[].label values match what appears in the spreadsheet
  • data_source.columns values match the spreadsheet header row exactly
  • Google Sheet is shared as "Anyone with the link can view"
  • TENANT_CONFIG_FILE in config.js points to the new file
