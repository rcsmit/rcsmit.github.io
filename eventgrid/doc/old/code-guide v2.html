The big picture
✦ Version 2 — 7 May 2026. The codebase has been refactored from a single HTML file into a proper multi-file JavaScript application. Each concern now lives in its own file. This guide reflects the new structure.

The app fetches events from a Cloudflare Worker (which reads a private Google Sheet), parses them into an array called rows, and renders them in one of three views: List, Calendar, or Map. Every filter change calls redraw(), which re-renders the active view using the current filter state.

There are three separate entry points, each a different HTML file:

FilePurpose
index.htmlFull application — List + Calendar + Map views, filter bar, all features. Embedded in WordPress via an iframe.
preview.htmlLightweight widget — card grid only, no filters, no map. Used for embedding a small events teaser elsewhere on the site.
show_event.htmlTest harness for the WordPress deeplink page. The actual WP page uses inline HTML, not this file.

When a filter changes — any pill click, dropdown change, date input, or search keystroke — it updates one or more state variables and calls redraw(). That's it. The rendering functions always read from visibleRows(), which applies all current filters every time it runs.

Script load order in index.html

Order matters — scripts rely on globals defined in earlier files:

#FileWhy it must come here
1config.jsDefines CONFIG and all state variables used by every other file.
2utils.jsPure helpers. Needed by data, list, map, modal.
3data.jsFetches data, defines rows, visibleRows(), redraw().
4map.jsUses rows, lColors, CONFIG.
5list.jsUses visibleRows(), openEventModal(), CONFIG.
6calendar.jsUses visibleRows(), openEventModal(), CONFIG.
7modal.jsUses lColors, CONFIG, rowID() from utils.

preview.html uses a subset. It only loads config.js, utils.js, list.js, and modal.js. It stubs out renderPagination(), visibleRows(), redrawMap(), renderCalendar(), and redraw() inline — because those live in files it doesn't load.

State variables

All global state is declared in config.js — the second half of the file, after the CONFIG object. Every render function reads from these. They update automatically when filters change.

VariableTypeWhat it holds
rowsArrayAll events loaded from the sheet. Each row is an object with spreadsheet column names as keys, plus _layer, _lat, _lon added by processData().
actLSetActive layers (continents). All layers are always active — leftover from earlier design. Safe to ignore.
actETSetActive event types (e.g. {"Festival","Retreat"}). Empty set = all types shown.
lColorsObjectMaps each continent name to a hex colour from CONFIG.COLORS.
qStrStringCurrent search query text.
dateFrom / dateToDate|nullActive date range filter. Null = no filter on that end.
hideEndedconst BooleanAlways true. Past events are always hidden.
filterContinent / filterCountryStringCurrently selected values. Empty string = no filter.
filterDealsBooleantrue = only show events that have an Insider Deal entry.
listPageNumberCurrent page number in list view. Resets to 1 on every redraw().
calYear / calMonthNumberCurrently displayed year and month in calendar view (month 0-indexed).
currentViewString'list', 'cal', or 'map'.
map / cluster / labelLayerLeaflet objectsOnly populated after the Map tab is clicked for the first time. null before that.
mapInitializedBooleanGuard flag — prevents Leaflet from initialising twice.
markerLayerMapWeakMapMaps each Leaflet marker object → its layer (continent) string. Used by cluster colouring.

5
Function reference

Click any function to see more detail. The file where each function lives is noted in the card.

🔵 Startup & Data — data.js
init()
Fetches data and boots the app
click to expand
The entry point. Tries the Cloudflare Worker first (5-second timeout). On failure, falls back to the public Google Sheets CSV URL. Both paths call processData(), then hideOv(). Called by app.js at the bottom of the script chain.
Calls: processData(), hideOv(), parseCSV(), setMsg()
processData(data)
Prepares raw rows and initialises the UI
click to expand
Adds _layer, _lat, _lon to each row. Assigns colours to layers. Builds the geo dropdowns, event type pills, and deals filter. Updates the header event count. Calls redraw().
Calls: assignColors(), buildGeoFilters(), buildEventTypes(), buildDealsFilter(), redraw()
redraw()
Central dispatcher — re-renders the active view
click to expand
Called after every filter change. Updates the "visible" count in the header, calls the appropriate render function for currentView, updates the ✕ Clear button, and calls parentIFrame.size() to notify the WordPress iframe resizer.
Calls: visibleRows(), renderList() or renderCalendar() or redrawMap(), updateClearButton()
🟣 Filtering — data.js
visible(row)
Returns true if a row passes all current filters
click to expand
The single source of truth for row visibility. Checks 8 conditions in order (see filter flow above). If something is showing or hiding unexpectedly, start here.
Calls: parseDate(), dateOnly()
visibleRows()
Returns filtered rows array
click to expand
Calls rows.filter(visible). Called by every render function. In preview.html, this is overridden with a stub that simply returns rows unfiltered (no filter UI in the preview).
buildDealsFilter()
Shows/hides the "Has deals" pill
click to expand
Checks if any non-ended row has an Insider Deal value. If none exist, the pill is hidden. If at least one exists, the pill appears. Called once by processData().
clearAllFilters()
Resets everything to the default state
click to expand
Clears dates, resets both geo dropdowns, clears actET, resets filterDeals, clears the search box, and calls redraw().
🟢 Rendering — list.js · calendar.js · app.js
renderList() list.js
Renders the event card grid
click to expand
Gets visibleRows(), sorts by start date, slices to the current page, builds event card elements. Calls renderCatPicks() first to show the ⭐ strip. If no events match, shows the "no events found" empty state. Calls renderPagination() at the end.
Calls: visibleRows(), renderCatPicks(), openEventOrNavigate(), renderPagination()
renderCatPicks(vis) list.js
Renders the ⭐ Cat's Picks horizontal strip
click to expand
Filters visible rows for those with STICKY IN MONTH VIEW set. Renders them as horizontally-scrolling cards above the main grid. Hidden when CAT_PICKS_STICKY = false or no shortlisted events are visible.
calcPageSize() list.js
Calculates how many cards fit on screen
click to expand
Reads the actual rendered width of the grid and calculates the column count using the same formula as the CSS auto-fill. Page size = columns × rows, minimum 9 cards total, minimum 3 rows. Re-calculated on every resize event.
renderCalendar() calendar.js
Renders the monthly calendar grid
click to expand
Builds a 7-column Mon–Sun grid for the current calYear/calMonth. For each day, finds all visible events spanning that day. Shows up to 3 coloured dots per cell; more → "+N more" popup. If no events in the month, shows an empty state message.
Calls: visibleRows(), parseDate(), openEventModal(), showCalPopupMulti()
setView(v) app.js
Switches between list, calendar, and map
click to expand
Shows/hides the three view containers, updates active state on toggle buttons, and triggers the appropriate render or initialisation.
🟠 Map — map.js
initMap()
Creates the Leaflet map — runs only once
click to expand
Sets up Leaflet map, tile layer, marker cluster group, and label layer. Cluster group uses a custom iconCreateFunction that colours clusters by dominant continent. Once done, calls redrawMap(). Guarded by mapInitialized.
redrawMap()
Clears and redraws all markers
click to expand
Clears all layers. Uses a jitter algorithm to spread markers that share the same coordinates (events in the same city) in a spiral so they don't pile on top of each other. Calls addMarker() for each visible row.
Calls: visibleRows(), addMarker(), syncLabels()
addMarker(row)
Adds one marker + label to the map
click to expand
Creates a dot marker. Binds a compact popup via mkMapPopupHtml(). Adds a text label to labelLayer (separate layer — putting labels in the cluster caused double dots, a previous bug). Shortlisted events get a larger, orange dot.
🟡 Modals & sharing — modal.js
openEventModal(row)
Opens the full event detail overlay
click to expand
Builds the event detail HTML and injects it into #event-modal-body, adds the open class, and locks page scroll. Includes WhatsApp share, Google Calendar, .ics download, and "Copy link" buttons. The deeplink URL uses rowID(row).
openEventOrNavigate(row) list.js
Opens modal OR navigates parent page
click to expand
Called from every card click. If EVENTS_DETAIL_URL is set to a non-empty string, it navigates window.parent to that URL + ?id=SLUG (for WordPress deeplink mode). Otherwise, opens the modal. Currently the modal is always used (EVENTS_DETAIL_URL = '').
shareWhatsApp(text)
Opens WhatsApp share cross-platform
click to expand
On iOS uses the native whatsapp:// scheme, falling back to wa.me if the app isn't installed. On other platforms opens wa.me directly. The share text includes event name, city, dates, and the shareable deeplink.
buildICS() / buildGCalURL()
Generates calendar export data
click to expand
buildICS() generates a valid RFC 5545 .ics file string with proper line folding. buildGCalURL() creates a Google Calendar "Add event" URL. Both treat events as all-day. Google Calendar end date is exclusive so +1 day is added. Both functions are also used by eventdetail-wp.js.
⚪ Utilities — utils.js
rowID(row)
Generates a stable slug for a row
click to expand
Returns a URL-safe slug like sofia-intensive-2026-05-09 from the event name and start date. Used for shareable deeplinks (?id=SLUG). Defined in both utils.js (used by modal.js) and eventdetail-wp.js (slightly different — strips trailing hyphens). The utils.js version is the canonical one.
parseDate(str)
Parses a date string into a JS Date
click to expand
Accepts YYYY-MM-DD (ISO, recommended) and MM/DD/YYYY (US). Returns null for empty or unparseable strings. Falls back to new Date(str) as a last resort.
escHtml(s)
Escapes a string for safe HTML insertion
click to expand
Replaces &, <, >, and " with HTML entities. Always use this when inserting spreadsheet data into HTML strings. Skipping it is an XSS risk and causes display bugs with special characters.
parseCSV(csv)
Parses raw CSV text into row objects
click to expand
RFC-compliant CSV parser handling quoted fields, escaped quotes, CRLF/LF/CR. Only active when the Worker fails and the app falls back to the public Google Sheets CSV URL.
assignColors(layers)
Maps each continent to a colour
click to expand
Assigns each continent name a colour from CONFIG.COLORS in alphabetical order, cycling if there are more than 30 continents. Result stored in the global lColors object.

CONFIG reference — config.js

The CONFIG object at the top of config.js is the only part you should need to change for routine maintenance. Everything else reads from these values.

KeyWhat it does
SHEET_ID_ENCGoogle Sheet ID — used only in the CSV fallback path.
SHEET_TAB_ENCSheet tab name — same note.
COL_*Exact column header names from the sheet. If a column is renamed in the sheet, update the matching COL_* here AND in the Worker's columns array.
MAP_CENTER / MAP_ZOOMWhere the map opens. Default: [20, 10] zoom 2.
SHOW_LABELSSet to false to hide event name labels on the map.
LABEL_MIN_ZOOMLabels only appear at this zoom level or higher. Default: 6.
CLUSTER_RADIUSHow close markers must be to cluster (pixels). Lower = more clusters.
CLUSTER_UNCLUSTRZoom level where clustering stops and all markers show individually. Default: 12.
COLORS30 fixed hex colours for continent→colour mapping. First colour = first continent alphabetically.
SL_COLORColour for shortlisted events (STICKY IN MONTH VIEW). Default: orange #f19072.
PAGE_SIZEMinimum events per page in list view. Actual page size is calculated dynamically by calcPageSize() based on screen width.

WordPress: setting up the show-event page

The show-event page is a standalone WordPress page that displays a single event's full details. It is reached via a shareable deeplink like:

https://www.acroinsiders.com/show-event/?id=sofia-intensive-2026-05-09

When someone clicks "Share on WhatsApp" or "Copy link" on an event, this is the URL they get. The page loads the full event dataset, finds the matching row by slug, and renders it without needing the full calendar app.

⚠️ Do not name the page "event". WordPress's The Events Calendar plugin reserves that slug. Use show-event or anything else that doesn't conflict.
Step-by-step setup in WordPress
  1. Go to Pages → Add New in the WordPress admin.
  2. Give the page any title, e.g. Event Detail. Set the slug (permalink) to show-event.
  3. Switch the editor to Code Editor mode (the three-dot menu → Code Editor, or click the HTML block button).
  4. Paste the following HTML block as the entire page content — replace any existing content:
Paste this into the WordPress Code Editor (HTML block) 
<div id="event-detail-root"></div>

<link rel="stylesheet" href="https://rcsmit.github.io/acroinsiders_events/css/style.css">

<style>
#event-modal {
  opacity: 1;
  pointer-events: all;
  position: relative;
  inset: auto;
  background: none;
  padding: 2rem 1rem;
}
</style>

<script src="https://rcsmit.github.io/acroinsiders_events/js/config.js"></script>
<script src="https://rcsmit.github.io/acroinsiders_events/js/utils.js"></script>
<script src="https://rcsmit.github.io/acroinsiders_events/js/modal.js"></script>
<script src="https://rcsmit.github.io/acroinsiders_events/js/eventdetail-wp.js"></script>
  1. Publish the page. The URL should be https://www.acroinsiders.com/show-event/.
  2. Test it by visiting:
    https://www.acroinsiders.com/show-event/?id=sofia-intensive-2026-05-09
    (replace the slug with a real event ID from your sheet).

What the show-event page renders

The rendered event detail includes everything from the modal, plus some extras specific to this standalone page:

  • Event name, category badge, continent badge
  • Date range, location, Google Maps link (estimated city-level)
  • ⭐ Shortlisted badge (if applicable)
  • 🏷️ Insider Deal block (if applicable)
  • ⚠️ "Event has already taken place" warning for past events
  • 🎪 "Event has already started" notice for ongoing events
  • Description paragraphs
  • Action buttons: Visit website, Add to Google Calendar, Download .ics, Share on WhatsApp, Copy link, ← Back to all events
How event slug IDs work

The slug is generated from the event name and start date:

// rowID() in utils.js
"Sofia Intensive" + "2026-05-09"
→ "sofia-intensive-2026-05-09"
→ URL: /show-event/?id=sofia-intensive-2026-05-09

Spaces and special characters become hyphens; uppercase becomes lowercase. The slug is stable as long as the event name and start date don't change in the sheet.

⚠️ Slug stability. If you rename an event or change its start date in the Google Sheet, any previously shared links will break — the old slug will no longer match any row. Avoid renaming events after they've been shared publicly.
Testing locally (show_event.html)

The file show_event.html in the repo is a local test harness. Open it in a browser with a URL parameter to test without WordPress:

file:///path/to/acroinsiders_events/show_event.html?id=sofia-intensive-2026-05-09

It loads the same JS files as the WordPress page but from the local js/ folder instead of GitHub Pages URLs. It is not deployed or used in production — it's for development only.

WordPress: main iframe (events calendar page)

The main events calendar (index.html) is embedded on the WordPress site as an iframe. The iframe auto-resizes to match its content height using the iframe-resizer library.

ℹ️ WordPress iframe embed snippet
Paste this into a WordPress HTML block on the events page: 
<iframe
  src="https://rcsmit.github.io/acroinsiders_events/index.html"
  id="events-frame"
  style="width:100%;border:none;display:block;"
  scrolling="no"
></iframe>
<script src="https://cdn.jsdelivr.net/npm/iframe-resizer@4.3.9/js/iframeResizer.min.js"></script>
<script>
  iFrameResize({ log: false, checkOrigin: false }, '#events-frame');
</script>

The iframe content (index.html) loads iframeResizer.contentWindow.min.js from its own <head>, and calls window.parentIFrame.size() after every redraw and on a timeout after load to ensure the height stays correct.

✓ The ALLOWED_ORIGINS list in worker.js must include the WordPress domain (https://acroinsiders.com) and the GitHub Pages domain (https://rcsmit.github.io) for CORS to pass. It currently does.
9
Common tasks
Add a new event type (e.g. "Workshop")

In data.js, find ET_ORDER and add the new type:

const ET_ORDER = ['Festival', 'Retreat', 'Intensive', 'Event', 'Training', 'Workshop'];

Then add it to etIcon() in utils.js:

function etIcon(t) {
  return { ..., 'Workshop': '🛠️' }[t] || '';
}

The name must match exactly what appears in the EVENT CATEGORY column in the sheet.

Add a new column / field to the event detail
  1. Add the column to the Google Sheet.
  2. Add the column name to the columns array in worker.js (the Cloudflare Worker), otherwise it will be stripped before reaching the app.
  3. Add a COL_* entry to CONFIG in config.js.
  4. In modal.js → openEventModal() extract the value: const myField = (row[CONFIG.COL_MYFIELD] || '').trim();
  5. Add it to the HTML template string in the same function.
  6. If it should also appear on the standalone WP page, repeat step 4–5 in eventdetail-wp.js → renderEventDetail().
Change the Worker URL

The Worker URL appears in three places — update all three if you change it:

  • data.js → inside init()
  • preview.html → inside initPreview()
  • eventdetail-wp.js → inside loadAndRender()
Add a new Worker secret (column group)

If you add a second sheet tab (e.g. venues), add a new route to the ROUTES object in worker.js, add the matching secrets in the Cloudflare dashboard, and fetch the new route from a new init function in the app.

Disable Cat's Picks strip

In list.js, set CAT_PICKS_STICKY = false. The strip will be hidden even if shortlisted events exist.

Switch event clicks from modal to WP deeplink

In list.js, change EVENTS_DETAIL_URL from '' to the WordPress show-event URL:

const EVENTS_DETAIL_URL = 'https://www.acroinsiders.com/show-event/';

Clicking a card will then navigate window.parent (the WP page) to the deeplink instead of opening the modal inside the iframe.

Things to watch out for
⚠️ Always use escHtml() when inserting sheet data into HTML.
Event descriptions can contain <, &, quotes, and other characters that break HTML. If you add new fields to the popup or cards, wrap them in escHtml(). Skipping this is an XSS risk and will cause display bugs.
⚠️ The map is null until the Map tab is clicked.
If you write code that references map, cluster, or labelLayer, check if (!mapInitialized) return; first. Otherwise you'll get a null reference error on page load.
⚠️ Labels go on labelLayer, not cluster.
Adding text labels to the cluster group causes each event to render as two dots. Text labels must always go on labelLayer.
⚠️ Worker strips columns.
The Cloudflare Worker only passes through the columns listed in its columns array. If you add a new column to the sheet and want to use it in the app, you must also add it to the Worker's route config — otherwise it arrives as an empty string.
⚠️ preview.html stubs override list.js functions.
preview.html redefines calcPageSize(), visibleRows(), renderPagination(), redraw(), and several others inline after loading list.js. The stub versions are simpler and don't need the full filter state. Be careful if you add new functions to list.js that preview.html calls — you may need to add stubs there too.
ℹ️ Date format matters.
parseDate() handles YYYY-MM-DD and MM/DD/YYYY. Stick to YYYY-MM-DD in the sheet — it's unambiguous and parses correctly across all browsers.
ℹ️ rowID() is defined twice — utils.js and eventdetail-wp.js.
Both generate the same slug format. The utils.js version is used in the main app; eventdetail-wp.js has its own copy because it doesn't load utils.js in all contexts. If you change the slug format, update both.
ℹ️ hideEnded is a constant.
Past events are always hidden. It is declared as const hideEnded = true. To make it toggleable, change it to let, add a UI toggle, and call redraw().
✓ Safe places to add things.
The easiest places to extend the app: renderList() in list.js (add something to cards), openEventModal() in modal.js (add something to the detail view), renderEventDetail() in eventdetail-wp.js (add something to the WP standalone page). All three are self-contained and follow a clear pattern.
✓ Finding functions fast.
Every section in the JavaScript files starts with a comment banner like /* ═══ SECTION NAME ═══ */. In VS Code, press Ctrl+Shift+F to search across all files.
