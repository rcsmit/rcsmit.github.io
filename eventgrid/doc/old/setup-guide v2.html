This guide walks you through every single step. No technical experience required.
You will need: a computer, a browser, and about 60–90 minutes.

At the end you will have:

  • A private Google Sheet only you can access
  • A live website at a free address like yourusername.github.io/acromap
  • A security layer (Cloudflare Worker) that keeps the sheet private

1
Google Account & Google Sheet
1.1 Create a Google account (skip if you have one)
  1. Go to accounts.google.com/signup
  2. Fill in your name, choose an email address ending in @gmail.com, set a password
  3. Follow the on-screen steps to verify your phone number
  4. You now have a Google account
1.2 Create your Google Sheet
  1. Go to sheets.google.com
  2. Click the big + button (Blank spreadsheet)
  3. Click "Untitled spreadsheet" at the top and rename it to Acromap Events
  4. In Row 1, type these column headers exactly — one per cell, starting at cell A1:
    ABCDEFGHIJK
    EVENT NAME EVENT START DATE EVENT END DATE EVENT CATEGORY EVENT DESCRIPTION CONTINENT COUNTRY STATE LAT LON STICKY IN MONTH VIEW
    ⚠️ Important: The capitalisation and spacing must be exact. Copy-paste each header from above.
  5. Add a few test events in the rows below. For EVENT CATEGORY use one of: Festival, Retreat, Intensive, Event
  6. For dates use the format 2026-06-15 (year-month-day)
  7. For LAT and LON use decimal coordinates — for example 48.8566 and 2.3522 for Paris
  8. Keep the sheet private — do not change its sharing settings
1.3 Copy your Sheet ID

Look at the URL in your browser. It looks like this:

https://docs.google.com/spreadsheets/d/1ABC123xyzYourSheetIDhere/edit

The long string between /d/ and /edit is your Sheet ID. Copy it and save it in a text file on your desktop — you will need it later.

2
Google Cloud (Service Account)

This is the part that lets a computer program read your private sheet without a password. A service account is like a robot user — you give it read-only access to your sheet, and it fetches the data for your map.

2.1 Go to Google Cloud Console
  1. Go to console.cloud.google.com
  2. Sign in with the same Google account you used for the sheet
  3. You may be asked to agree to terms of service — click Agree and Continue
2.2 Create a Project
  1. At the top of the page, click the dropdown that says Select a project
  2. Click New Project
  3. Name it acromap (or anything you like)
  4. Click Create
  5. Wait a few seconds, then click Select Project in the notification that appears
2.3 Enable the Google Sheets API
  1. In the search bar at the top of the page, type Google Sheets API and click it in the results
  2. Click the blue Enable button
  3. Wait about 10 seconds for it to activate
2.4 Create a Service Account
  1. In the left sidebar, click IAM & Admin → Service Accounts
    If you do not see the sidebar, click the ☰ menu icon at the top left
  2. Click + Create Service Account
  3. For "Service account name" type: acromap-reader
  4. The "Service account ID" fills in automatically — leave it as-is
  5. Click Create and Continue
  6. On the "Grant this service account access" step — click Continue (skip it)
  7. On the "Grant users access" step — click Done (skip it)
  8. You now see your service account in the list. It has an email address like:
    acromap-reader@acromap-12345.iam.gserviceaccount.com
    Copy this email address and save it in your text file.
2.5 Create and download the Key
  1. Click on the service account email in the list to open it
  2. Click the Keys tab
  3. Click Add Key → Create new key
  4. Select JSON and click Create
  5. A JSON file downloads to your computer automatically. Keep this file safe — it is like a password. Do not share it or upload it anywhere.
  6. Open the file in a text editor (Notepad on Windows, TextEdit on Mac)
  7. Find the line that starts with "private_key": — it contains a long block starting with -----BEGIN PRIVATE KEY-----.
    Copy the entire value between the outer quotes, including the BEGIN and END lines.
    Save this in your text file — you will need it in Part 4.
⚠️ Security reminder: Never share the private key or paste it into an email, chat, or document. It goes only into the Cloudflare Worker secrets in Part 4.
2.6 Share your Google Sheet with the Service Account
  1. Go back to your Google Sheet
  2. Click the green Share button (top right)
  3. In the "Add people and groups" field, paste the service account email address (the one ending in .iam.gserviceaccount.com)
  4. Make sure the role is set to Viewer
  5. Uncheck "Notify people" — the robot does not have an inbox
  6. Click Share
✓ What you have now: A private sheet that only your service account (and you) can read. The sheet itself is not public.
3
GitHub Account & Repository

GitHub is where your website files live. GitHub Pages hosts the map for free at a public web address.

3.1 Create a GitHub account
  1. Go to github.com/signup
  2. Enter your email, create a password, choose a username
    Your username becomes part of your website address: username.github.io/acromap
  3. Follow the steps to verify your email and complete sign-up
3.2 Create a new repository

A repository (repo) is a folder on GitHub that holds your website files.

  1. Once logged in, click the + button at the top right → New repository
  2. For "Repository name" type: acromap
  3. Make sure it is set to Public (required for free GitHub Pages)
  4. Check the box Add a README file
  5. Click Create repository
3.3 Upload index.html
  1. You should now see your empty repository page
  2. Click Add file → Upload files
  3. Drag your index.html file onto the upload area (or click "choose your files")
  4. At the bottom you will see a "Commit changes" section:
    1. In the first text field (which says "Add files via upload") type a short description, for example: Add map page
    2. Leave everything else as-is
  5. Click the green Commit changes button
What is a commit message? It is just a short note describing what you changed — like a label in a filing cabinet. It helps you find changes later. Always write something brief and meaningful.
3.4 Enable GitHub Pages
  1. In your repository, click Settings (the tab with a gear icon ⚙)
  2. In the left sidebar, click Pages
  3. Under "Source", click the dropdown that says None and select main
  4. Leave the folder as / (root)
  5. Click Save
  6. Wait about 1–2 minutes, then refresh the page
  7. You will see a green box: "Your site is live at https://yourusername.github.io/acromap"
  8. Copy that URL and save it — this is your website address
4
Cloudflare Worker (the security layer)

The Worker sits between your website and Google Sheets. It reads your private sheet using the service account key, and passes the data to your map. Your sheet never becomes public.

4.1 Create a Cloudflare account
  1. Go to dash.cloudflare.com/sign-up
  2. Enter your email and create a password
  3. Verify your email address
4.2 Create a Worker
  1. In the left sidebar, click Workers & Pages
  2. Click Create → Create Worker
  3. Give it any name, for example: acromap-worker
  4. Click Deploy (ignore the default code for now)
  5. Click Edit code (the blue button)
4.3 Paste the Worker code
  1. Select all the existing code in the editor (Ctrl+A on Windows, Cmd+A on Mac) and delete it
  2. Copy the entire code block below and paste it into the editor: Worker code — paste this into the Cloudflare editor 
    const ROUTES = {
  '/events': {
    sheetIdEnv:  'EVENTS_SHEET_ID',
    sheetTabEnv: 'EVENTS_SHEET_TAB',
    columns: [
      'EVENT NAME',
      'EVENT START DATE',
      'EVENT END DATE',
      'EVENT CATEGORY',
      'EVENT DESCRIPTION',
      'CONTINENT',
      'COUNTRY',
      'STATE',
      'LAT',
      'LON',
      'STICKY IN MONTH VIEW',
    ],
  },
};

const ALLOWED_ORIGINS = [
  'https://YOURUSERNAME.github.io',   // ← replace with your GitHub username
  'http://localhost',
  'null',
];

const CACHE_SECONDS = 300;

export default {
  async fetch(request, env) {
    const origin  = request.headers.get('Origin') || '';
    const allowed = ALLOWED_ORIGINS.includes('*') || ALLOWED_ORIGINS.includes(origin);
    if (request.method === 'OPTIONS') {
      return new Response(null, { headers: corsHeaders(origin, allowed) });
    }
    if (request.method !== 'GET') {
      return new Response('Method not allowed', { status: 405 });
    }
    const path  = new URL(request.url).pathname;
    const route = ROUTES[path];
    if (!route) {
      return new Response(JSON.stringify({ error: `Unknown route "${path}"` }),
        { status: 404, headers: { 'Content-Type': 'application/json', ...corsHeaders(origin, allowed) } });
    }
    try {
      const sheetId  = env[route.sheetIdEnv];
      const sheetTab = env[route.sheetTabEnv];
      if (!sheetId || !sheetTab) throw new Error(`Missing secret: ${route.sheetIdEnv}`);
      const rows     = await fetchSheetRows(env, sheetId, sheetTab);
      const filtered = rows.map(row => filterRow(row, route.columns));
      return new Response(JSON.stringify({ rows: filtered, count: filtered.length }), {
        headers: { 'Content-Type': 'application/json',
          'Cache-Control': `public, max-age=${CACHE_SECONDS}`, ...corsHeaders(origin, allowed) },
      });
    } catch (err) {
      return new Response(JSON.stringify({ error: err.message }), {
        status: 500, headers: { 'Content-Type': 'application/json', ...corsHeaders(origin, allowed) },
      });
    }
  },
};

function corsHeaders(origin, allowed) {
  return {
    'Access-Control-Allow-Origin':  allowed ? origin || '*' : 'null',
    'Access-Control-Allow-Methods': 'GET, OPTIONS',
    'Access-Control-Allow-Headers': 'Content-Type',
  };
}

async function fetchSheetRows(env, sheetId, sheetTab) {
  const token = await getAccessToken(env.GOOGLE_SA_EMAIL, env.GOOGLE_SA_KEY);
  const url   = `https://sheets.googleapis.com/v4/spreadsheets/${sheetId}/values/${encodeURIComponent(sheetTab)}`;
  const res   = await fetch(url, { headers: { 'Authorization': 'Bearer ' + token } });
  if (!res.ok) { const t = await res.text(); throw new Error(`Sheets API ${res.status}: ${t.slice(0,200)}`); }
  const data   = await res.json();
  const values = data.values;
  if (!values || values.length < 2) throw new Error('Sheet is empty or has no data rows');
  const headers = values[0].map(h => h.trim());
  return values.slice(1).map(cols => {
    const obj = {};
    headers.forEach((h, i) => { obj[h] = (cols[i] || '').trim(); });
    return obj;
  });
}

function filterRow(row, columns) {
  const out = {};
  columns.forEach(col => { out[col] = row[col] || ''; });
  return out;
}

async function getAccessToken(clientEmail, privateKeyPem) {
  const now   = Math.floor(Date.now() / 1000);
  const claim = { iss: clientEmail,
    scope: 'https://www.googleapis.com/auth/spreadsheets.readonly',
    aud: 'https://oauth2.googleapis.com/token', iat: now, exp: now + 3600 };
  const jwt = await signJWT(claim, privateKeyPem);
  const res = await fetch('https://oauth2.googleapis.com/token', {
    method: 'POST',
    headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
    body: new URLSearchParams({ grant_type: 'urn:ietf:params:oauth:grant-type:jwt-bearer', assertion: jwt }),
  });
  const json = await res.json();
  if (!json.access_token) throw new Error('Token error: ' + JSON.stringify(json));
  return json.access_token;
}

async function signJWT(payload, pemKey) {
  const header  = { alg: 'RS256', typ: 'JWT' };
  const enc     = v => btoa(JSON.stringify(v)).replace(/\+/g,'-').replace(/\//g,'_').replace(/=/g,'');
  const message = enc(header) + '.' + enc(payload);
  const key     = await importPrivateKey(pemKey);
  const sig     = await crypto.subtle.sign({ name: 'RSASSA-PKCS1-v1_5' }, key, new TextEncoder().encode(message));
  const sigB64  = btoa(String.fromCharCode(...new Uint8Array(sig))).replace(/\+/g,'-').replace(/\//g,'_').replace(/=/g,'');
  return message + '.' + sigB64;
}

async function importPrivateKey(pem) {
  const b64    = pem.replace(/-----BEGIN PRIVATE KEY-----/, '')
                    .replace(/-----END PRIVATE KEY-----/, '')
                    .replace(/\s+/g, '');
  const binary = Uint8Array.from(atob(b64), c => c.charCodeAt(0));
  return crypto.subtle.importKey('pkcs8', binary.buffer,
    { name: 'RSASSA-PKCS1-v1_5', hash: 'SHA-256' }, false, ['sign']);
}

4.4 Add the Secret variables

Now you give the Worker the credentials it needs — stored securely, not written in the code.

  1. Go back to your Worker overview (click the Worker name at the top)
  2. Click Settings → Variables
  3. Scroll down to Environment Variables and click Add variable
  4. Add each of these four variables. Click "Encrypt" on each one before saving:
    Variable nameValue to paste
    GOOGLE_SA_EMAILThe service account email (ends in .iam.gserviceaccount.com)
    GOOGLE_SA_KEYThe full private key from the JSON file — the entire block from -----BEGIN PRIVATE KEY----- to -----END PRIVATE KEY-----
    EVENTS_SHEET_IDYour Sheet ID (the long string from the Google Sheet URL)

4.5 Find and test your Worker URL

  2. You will see a URL like https://acromap-worker.YOURNAME.workers.dev — copy and save it

Connect the map to the Worker

Now you update index.html so it loads data from your Worker instead of directly from Google Sheets.

5.1 Edit index.html

Open index.html in a text editor. Near the very bottom, find this function:

async function init() {
  setMsg('Fetching event data…');
  const csvURL=`https://docs.google.com/spreadsheets/d/...

Replace the entire init() function with this:

Replacement init() function for index.html 
async function init() {
  setMsg('Fetching event data…');
  let rawData;
  try {
    const res = await fetch('https://YOUR-WORKER-URL.workers.dev/events');
    if (!res.ok) throw new Error(`HTTP ${res.status}`);
    const json = await res.json();
    rawData = json.rows;
  } catch(e) {
    setMsg('❌ Could not load data: ' + e.message);
    setStep('Check the Worker URL in index.html');
    return;
  }
  if (!rawData || !rawData.length) { setMsg('No data found.'); return; }
  processData(rawData);
  hideOv();
}

Replace https://YOUR-WORKER-URL.workers.dev/events with your actual Worker URL followed by /events.

6
Adding and editing events

From now on, all you need to do is:

  1. Open your Google Sheet at sheets.google.com
  2. Add or edit rows below the header row
  3. The map updates automatically within 5 minutes
Note on past events: Events whose end date has passed are automatically hidden from the map. You do not need to delete them from the sheet.
7
Getting updates from the developer

The developer works on their own copy of the code (called a fork) and will send you improvements from time to time. Here is how to pull those updates into your repository.

What is a fork?

The developer has their own copy of index.html on GitHub. When they improve the map, you copy their new version into your repository. GitHub makes this easy — no tools needed.

7.1 The developer sends you a Pull Request

The developer will make their changes and then submit a Pull Request — a request for you to accept their changes into your repository.

You will receive an email from GitHub saying something like:

"Developer Name wants to merge changes into your repository"
7.2 Review and accept the update
  1. Click the link in the email — or go to your repository on GitHub and click Pull requests
  2. Click on the pull request title to open it
  3. Scroll down to see what changed (green lines = added, red lines = removed)
  4. If you are happy with the update, click the green Merge pull request button
  5. A confirmation box appears:
    1. Type a short message like: Accept map update
    2. Click Confirm merge
  6. GitHub Pages will automatically update your live site within 1–2 minutes
✓ That is all. You do not need to download anything or use any tools.
7.3 If the developer sends you a file directly

If the developer sends you a new index.html by email or file transfer instead of a Pull Request:

  1. Go to your repository on GitHub
  2. Click Add file → Upload files
  3. Drag the new index.html onto the upload area (it will replace the old one automatically)
  4. In the commit message field type: Update map from developer
  5. Click the green Commit changes button
7.4 After any update — check your Worker URL

After accepting an update, quickly verify that your Worker URL is still in the file:

  1. In your repository, click on index.html to open it
  2. Press Ctrl+F (or Cmd+F on Mac) to open the search box
  3. Search for workers.dev — you should see your Worker URL
  4. If it is missing, click the pencil icon ✏️ to edit the file and restore your URL:
    1. Find the line: fetch('https://...workers.dev/events')
    2. Replace the URL with your own Worker URL
    3. Type a commit message: Restore Worker URL
    4. Click Commit changes

Troubleshooting
❌ The Worker returns an error about "Token error"
The GOOGLE_SA_KEY secret is wrong. Make sure you copied the entire private key including the -----BEGIN PRIVATE KEY----- and -----END PRIVATE KEY----- lines. Some text editors add extra line breaks — try pasting it again carefully.
❌ The Worker returns "Sheets API 403"
You forgot to share the Google Sheet with the service account email. Go back to step 2.6 and share it.
❌ The Worker returns "Sheet is empty or has no data rows"
The EVENTS_SHEET_TAB secret is wrong. Check the tab name at the very bottom of your Google Sheet. It is probably Sheet1 unless you renamed it. Update the secret in Cloudflare to match exactly.
❌ The map shows "Could not load data"
The Worker URL in index.html is wrong. Double-check that you replaced YOUR-WORKER-URL with your real Cloudflare Worker address.
❌ GitHub Pages shows a blank page or an old version
GitHub Pages can take 2–5 minutes to update after you commit a file. Wait a few minutes and do a hard refresh (Ctrl+Shift+R or Cmd+Shift+R).
❌ The map loads but shows no events
Check that your dates are in YYYY-MM-DD format (for example 2026-06-15). Events whose end date has already passed are hidden by default.
❌ The map shows events but the pins are all in the ocean
The LAT and LON values in your sheet are missing or formatted incorrectly. They must be decimal numbers like 48.8566 (not "48° 51' N").
