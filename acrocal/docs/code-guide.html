The big picture

The entire application is a single HTML file — no build step, no npm, no framework. Open it in a browser and it works. This keeps things simple but means everything lives in one place.

The app fetches events from a Cloudflare Worker (which reads a private Google Sheet), parses them into an array called rows, and then renders them in one of three views: List, Calendar, or Map. Every filter change just calls redraw(), which re-renders the active view using the current filter state.

User sees the page

When a filter changes — any pill click, dropdown change, date input, or search keystroke — it updates one or more state variables and calls redraw(). That's it. The rendering functions always read from visibleRows(), which applies all current filters every time it runs.

State variables

These are the global variables that hold the application's current state. Every render function reads from these. You don't usually need to touch them — they update automatically when filters change.

VariableTypeWhat it holds
rowsArrayAll events loaded from the sheet. Each row is an object with spreadsheet column names as keys, plus _layer, _lat, _lon added by processData().
actLSetActive layers (continents). Currently all layers are always active — this is a leftover from an earlier design. Safe to ignore.
actETSetActive event types (e.g. {"Festival", "Retreat"}). Empty set = all types shown.
lColorsObjectMaps each continent name to a hex colour from CONFIG.COLORS.
qStrStringCurrent search query text.
dateFrom / dateToDate or nullActive date range filter. Null means no filter on that end.
hideEndedBoolean (const)Always true. Past events are always hidden. To change this behaviour, turn it into a variable and add a toggle.
filterContinent / filterCountryStringCurrently selected continent/country. Empty string = no filter.
listPageNumberCurrent page number in list view. Resets to 1 on every redraw().
calYear / calMonthNumberCurrently displayed year and month in calendar view (month is 0-indexed: 0=Jan, 11=Dec).
currentViewString'list', 'cal', or 'map'.
map / cluster / labelLayerLeaflet objectsOnly populated after the Map tab is clicked for the first time. null before that.
mapInitializedBooleanGuard flag — prevents Leaflet from being initialised twice.

What visibleRows() checks — in order

1Layer (continent) is activeactL
2Event has not ended (end date ≥ today)hideEnded (always true)

5
Function reference

Click any function to see more detail. Grouped by what they do.

🔵 Startup & Data
init()
Fetches data and boots the app
click to expand
The entry point. Fetches JSON from the Cloudflare Worker URL, hands the rows array to processData(), then removes the loading overlay. If the fetch fails, it shows an error on the loading screen and stops. Called once automatically at the bottom of the script.
Calls: processData(), hideOv(), setMsg()
processData(data)
Prepares raw rows and initialises the UI
click to expand
Takes the raw array from the Worker. Adds _layer, _lat, and _lon as private properties to each row. Assigns colours to layers, builds the dropdowns, creates the event type pills, updates the header count, and calls redraw().
Calls: assignColors(), buildGeoFilters(), buildEventTypes(), rowsForDropdowns(), redraw()
parseCSV(csv)
Parses a raw CSV string into an array of objects
click to expand
A hand-written RFC-compliant CSV parser. Handles quoted fields, escaped quotes, and Windows/Unix line endings. Returns an array of objects where each key is a column header. Only used when fetching directly from a public Google Sheet — currently not in use since the Worker returns JSON.
assignColors(layers)
Maps each continent to a colour
click to expand
Goes through the sorted list of continent names and assigns each one a colour from CONFIG.COLORS, cycling back to the start if there are more continents than colours. Result stored in the global lColors object.
🟣 Filtering
visible(row)
Returns true if a row should be shown
click to expand
The single source of truth for whether a row is visible. Checks seven conditions in order (see the filter flow diagram above). The most important function in the filtering system — if something is showing or hiding unexpectedly, start here.
Calls: parseDate(), dateOnly()
visibleRows()
Returns all rows that pass the current filters
click to expand
Simply calls rows.filter(visible). Every render function calls this at the start. Called frequently — keep visible() fast.
rowsForDropdowns()
Rows for building continent/country options
click to expand
Like visibleRows() but only applies hideEnded and actET — not geo or search filters. This is what the continent and country dropdowns should be based on: they should only show locations that have at least one current, matching-type event, regardless of any geo or search filter the user has set.
refreshGeoDropdowns()
Rebuilds continent/country dropdowns, keeping selections if still valid
click to expand
Called when an event type pill is toggled. Rebuilds both dropdowns from rowsForDropdowns(). If the previously selected continent or country still exists in the new list, it keeps the selection — otherwise it resets to "All".
Calls: rowsForDropdowns()
buildGeoFilters()
Builds continent/country dropdowns on initial load
click to expand
Called once by processData(). Similar to refreshGeoDropdowns() but simpler — no need to preserve selections since everything is fresh.
onContinentChange()
User changed the continent dropdown
click to expand
Updates filterContinent, resets filterCountry, and rebuilds the country dropdown to only show countries within the selected continent. Then calls redraw().
buildEventTypes()
Creates the event type pill buttons in the filter bar
click to expand
Reads from ET_ORDER (the ordered list of type names) and ET_PREMIUM (locked types). Creates a <span> element for each type, adds a click listener that toggles the type in actET, then calls refreshGeoDropdowns() and redraw(). To add a new event type, add it to ET_ORDER.
clearAllFilters()
Resets everything to the default state
click to expand
Clears dates, resets both dropdowns, clears actET, clears the search box, and calls redraw().
🟢 Rendering
redraw()
Central dispatcher — re-renders the active view
click to expand
The central function called after every filter change. Updates the "visible" count in the header, then calls the appropriate render function for currentView. Always resets listPage to 1 so the user doesn't end up on a page that no longer exists.
Calls: visibleRows(), renderList() or renderCalendar() or redrawMap()
setView(v)
Switches between list, calendar, and map views
click to expand
Shows/hides the three view containers, updates the active state on the toggle buttons, and triggers the appropriate render or initialisation for the new view.
renderList()
Renders the event cards in list view
click to expand
Gets visibleRows(), slices to the current page, and builds an event card element for each row. If no events match, shows the "no events found" message. Calls renderPagination() at the end.
Calls: visibleRows(), fmtDateRange(), openEventModal(), renderPagination()
renderPagination(pages, total)
Builds the pagination bar below the list
click to expand
Renders prev/next buttons, numbered page buttons (with smart ellipsis for large page counts), and a "X–Y of Z" info label. Hidden when there is only one page. Page size is set by CONFIG.PAGE_SIZE.
renderCalendar()
Renders the monthly calendar grid
click to expand
Builds a 7-column grid for the current calYear / calMonth. For each day, finds all visible events that span that day and renders coloured dots (max 3 per day, then a "+N more" link). Multi-day events appear on every day they span. Clicking a dot opens the event detail modal. If no events are visible at all, shows a friendly "no events" message instead of an empty grid.
Calls: visibleRows(), parseDate(), openEventModal(), showCalPopupMulti()
🟠 Map
initMap()
Creates the Leaflet map — runs only once
click to expand
Sets up the Leaflet map, tile layer, marker cluster group, and label layer. The cluster group has a custom iconCreateFunction that colours clusters by the dominant continent. Once done, calls redrawMap(). The mapInitialized flag stops this running again on subsequent visits to the Map tab.
redrawMap()
Clears and redraws all markers
click to expand
Clears all layers and calls addMarker() for every visible row. Also calls syncLabels() to show/hide text labels based on zoom level.
Calls: visibleRows(), addMarker(), syncLabels()
addMarker(row)
Adds one marker + label to the map
click to expand
Creates a dot marker at the row's lat/lon using mkDotIcon(). Binds a compact popup using mkMapPopupHtml(). Also adds a text label to labelLayer (separate from the cluster layer — that was a previous bug that caused double dots). Shortlisted events get a larger, orange dot.
Calls: mkDotIcon(), mkMapPopupHtml()
mkMapPopupHtml(row)
Builds the compact HTML for a map marker popup
click to expand
Returns an HTML string with the event name, location, date, and a "View details →" link. Clicking the name or the link calls openEventFromMap(idx) which opens the full event detail modal. The idx is the row's index in the rows array — used because Leaflet popup click handlers run in global scope.
syncLabels()
Shows or hides text labels based on zoom
click to expand
Text labels are always in the DOM but hidden at low zoom. This function runs on every zoomend event. Labels appear when zoom ≥ CONFIG.LABEL_MIN_ZOOM (default: 6).
🟡 Modals & popups
openEventModal(row)
Opens the full event detail overlay
click to expand
Builds the HTML content for the event detail modal from the row's data, injects it into #event-modal-body, adds the open class, and locks page scroll. The description is split on double line breaks to create paragraphs. Called from card clicks in list view, dot clicks in calendar, and "View details" in map popups.
showCalPopupMulti(dayEvents, day, e)
Shows the floating "+N more" popup on calendar
click to expand
Triggered when a day has more than 3 events. Positions itself near the click coordinates, adjusting if it would go off-screen. Clicking an event name inside the popup opens the full event modal and closes the popup.
⚪ Utilities & helpers
parseDate(str)
Parses a date string into a JS Date object
click to expand
Accepts two formats: YYYY-MM-DD (ISO, recommended) and MM/DD/YYYY (US). Returns null for empty or unparseable strings. Falls back to new Date(str) for other formats.
dateOnly(d)
Converts a Date to a comparable integer
click to expand
Returns an integer like 20260615 (YYYYMMDD). Used to compare dates without time-of-day complications. Two dates on the same day are equal even if they have different hours.
fmtDateRange(row)
Formats the start–end date as a readable string
click to expand
Returns something like "16 May 2026 – 18 May 2026", or just "16 May 2026" for single-day events. Returns an empty string if both dates are missing.
escHtml(s)
Escapes a string for safe HTML insertion
click to expand
Replaces &, <, >, and " with HTML entities. Always use this when inserting spreadsheet data into HTML strings — event descriptions and names can contain any character.
hexToRgba(hex, alpha)
Converts a hex colour to rgba
click to expand
Used by the map cluster icons to create semi-transparent versions of continent colours. For example hexToRgba('#f19072', 0.18) gives the pale background of a cluster bubble.

CONFIG reference

The CONFIG object at the top of the script is the only part you should need to change for routine maintenance. Everything else is wired to use these values.

KeyWhat it does
SHEET_ID_ENCThe Google Sheet ID — only used if the Worker URL is removed and data is loaded directly (public sheet mode). Currently inactive.
SHEET_TAB_ENCThe tab name — same note as above.
COL_*The exact column header names from the sheet. If a column is renamed in the sheet, update the matching COL_* here.
MAP_CENTER / MAP_ZOOMWhere the map opens. Default is [20, 10] at zoom 2 — roughly centred on Europe/Africa.
SHOW_LABELSSet to false to hide event name labels on the map entirely.
LABEL_MIN_ZOOMLabels only appear at this zoom level or above. Default: 6.
CLUSTER_RADIUSHow close markers need to be to cluster together (pixels). Lower = more clusters.
CLUSTER_UNCLUSTRZoom level at which clustering stops and all individual markers show. Default: 12.
COLORS30 hex colours used for continent-to-colour mapping. First colour = first continent alphabetically.
SL_COLORColour used for shortlisted events (the STICKY IN MONTH VIEW column). Default: orange #f19072.
PAGE_SIZENumber of events per page in list view. Default: 10.

Common tasks
Add a new event type (e.g. "Workshop")

Find ET_ORDER near the top of the event type filter section:

const ET_ORDER = ['Festival', 'Retreat', 'Intensive', 'Event', 'Teacher Training'];

Add your new type to the array. Then add it to the etIcon() function to give it an emoji:

function etIcon(t) {
  return {'Festival':'🎪', 'Retreat':'🏕️', ... , 'Workshop':'🛠️'}[t] || '';
}

Make sure the name matches exactly what appears in the EVENT CATEGORY column in the sheet.

Change the number of events per page

In CONFIG, change PAGE_SIZE: 10 to whatever number you want.

Add a field to the event detail popup

The popup is built in openEventModal(row). Each row property is accessed as row[CONFIG.COL_SOMETHING]. To add a new field:

  1. Add a new COL_* entry in CONFIG pointing to the sheet column name
  2. In openEventModal(), extract it: const myField = (row[CONFIG.COL_MYFIELD] || '').trim();
  3. Add it to the HTML template string in the same function
  4. Add the column name to the columns array in the Cloudflare Worker's ROUTES config, otherwise the Worker will strip it out
Change the map's starting position

In CONFIG: MAP_CENTER: [lat, lon] and MAP_ZOOM: 2. For example, to open centred on Europe: MAP_CENTER: [52, 10], MAP_ZOOM: 4.

Change the Worker URL

Find the init() function at the bottom of the script. Change the URL in the fetch() call:

const res = await fetch('https://YOUR-WORKER.workers.dev/events');
8
Things to watch out for
⚠️ Always use escHtml() when inserting sheet data into HTML.
Event descriptions can contain <, &, quotes, and other characters that break HTML. If you add new fields to the popup or cards, wrap them in escHtml(). Skipping this is an XSS risk and will also cause display bugs.
⚠️ The map is null until the Map tab is clicked.
If you write code that references map, cluster, or labelLayer outside of initMap() or redrawMap(), check if (!mapInitialized) return; first. Otherwise you'll get a null reference error when the page loads in list or calendar view.
⚠️ Labels go on labelLayer, not cluster.
This was a previous bug — adding text label markers to the cluster group caused each event to render as two dots. Text labels must always go on labelLayer.
ℹ️ The Worker strips columns.
The Cloudflare Worker only passes through the columns listed in its columns array. If you add a new column to the sheet and want to use it in the app, you must also add it to the Worker's route config — otherwise it arrives as an empty string.
ℹ️ Date format matters.
parseDate() handles YYYY-MM-DD and MM/DD/YYYY. Other formats will fall back to new Date(str) which behaves inconsistently across browsers. Stick to YYYY-MM-DD in the sheet.
ℹ️ hideEnded is a constant, not a variable.
Past events are always hidden. It is declared as const hideEnded = true. If you want to make it toggleable, change it to let hideEnded = true and add a checkbox or button that changes it and calls redraw().
✓ Safe places to add things.
The easiest place to add a new feature is in renderList() (to add something to the cards) or openEventModal() (to add something to the detail view). Both are self-contained and follow a clear pattern. The filter system is slightly more complex — if you add a new filter, you need to add it to visible(), initialise it in the state variables, and wire up a UI element that updates the state and calls redraw().
